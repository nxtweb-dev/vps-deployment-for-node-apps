# Contributing to VPS Deployment for Node.js Apps

## How to contribute?

### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Clearly describe the problem, including steps to reproduce it and your environment (OS version, Node version).

### Suggesting Enhancements
* Open an issue labeled "enhancement" to discuss your idea.
* Be specific about why this change would be helpful for others.

### Pull Requests
1. **Fork** the repository.
2. **Clone** your fork to your local machine.
3. Create a **branch** for your feature or fix (`git checkout -b feature/my-new-feature`).
4. **Commit** your changes with clear, descriptive messages.
5. **Push** to your fork and submit a **Pull Request**.

## Style Guide
* Use clear, concise Markdown.
* Ensure all shell commands are tested on a clean Ubuntu/Debian environment.
* Keep Nginx configurations modular and well-commented.
